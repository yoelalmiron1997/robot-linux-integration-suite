Robot Framework Linux Integration Testing

Automated validation of software distributed as Debian (.deb) packages on Linux environments.

Robot Framework Python 3 Linux Debian (.deb) dpkg / apt Docker GitHub Actions

1. Problem & Context

Software deployed to enterprise Linux systems is frequently packaged as Debian binaries (.deb). Validating these packages requires more than standard API or web UI testing: it demands verifying package installation states, pre/post installation scripts, file structure placement, service lifecycle execution, configuration schema persistence across upgrades, and clean uninstallation without orphan files.

This project demonstrates an end-to-end integration testing suite built around Robot Framework and Python to validate a synthetic daemon named satellite-telemetry across its entire lifecycle on Linux.

2. Test System Architecture

The system under test is packaged as Debian binaries (v1.0.0 and v1.1.0) and tested in an isolated Debian Linux container environment.

3. Test Strategy & Custom Library

Robot Framework acts as the core orchestration engine. To avoid messy bash commands embedded inside test cases, reusable keywords are abstracted across specialized resource files and a custom Python library (LinuxPackageLibrary.py).

The custom library extends Robot Framework to query package databases via dpkg-query, parse JSON configuration files, inspect system log files, and verify running process PIDs.

5. Execution & Reproducibility

The entire suite is fully reproducible locally via Docker or in continuous integration via GitHub Actions:
